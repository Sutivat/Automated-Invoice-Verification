# 📑 Typhoon Invoice AI: Smart Finance Verifier

An intelligent, batch-processing web application designed to automate invoice verification using **Typhoon OCR** and **Typhoon LLM**. This tool bridges the gap between raw document scanning and structured financial approval, featuring a **Human-in-the-Loop (HITL)** interface to ensure 100% data accuracy before exporting to ERP systems.

---

## 🚀 Key Features

* **Advanced 2-Way Matching**: Automatically validates invoice totals against a mock ERP database (`DATABASE_PO`) to ensure financial accuracy.
* **Duplicate Detection**: Guards against double-billing by checking against a `PAID_INVOICES` registry.
* **Rich Data Extraction**: Extracts granular details including **Tax ID**, **Invoice Date**, **Subtotal**, and **VAT (7%)**.
* **Dual-Mode Processing**:
    * **Local (Ollama)**: Uses `typhoon-ocr1.5-3b` and `typhoon2.5-4b` for high privacy.
    * **Cloud (OpenTyphoon API)**: Uses `typhoon-v2.5-30b` for superior agentic reasoning.
* **Human-in-the-Loop (HITL)**: A specialized AP review table with editable actions (**Approve**, **Hold**, **Reject**) and automated remarks.
* **ERP-Ready Export**: One-click CSV export with standardized column headers (e.g., `DocDate`, `InvoiceNo`, `TaxAmount`) and `utf-8-sig` encoding for Thai character support in Excel.

---

## 🛠️ Tech Stack

* **Frontend**: [Streamlit](https://streamlit.io/)
* **OCR Engine**: [Typhoon OCR](https://opentyphoon.ai/) (v1.5-3b local / Cloud API)
* **Reasoning Engine**: [Typhoon 2.5 LLM](https://opentyphoon.ai/) (Ollama or Cloud API)
* **Data Handling**: Pandas & Python-Dotenv

## 📋 Installation

1.  **Clone the repository**:
    ```bash
    git clone https://github.com/Sutivat/Automated-Invoice-Verification.git
    cd Automated-Invoice-Verification
    ```

2.  **Install dependencies**:
    ```bash
    pip install -r requirements.txt
    ```

3.  **Setup Environment Variables**:
    Create a `.env` file in the root directory and add your Typhoon API key:
    ```env
    TYPHOON_API_KEY=your_api_key_here
    ```

---

## 🚦 Usage

1.  **Start the Application**:
    ```bash
    streamlit run demo_for_invoice.py
    ```

2.  **Configuration**:
    * Use the **Sidebar** to choose between **LOCAL** (requires Ollama running with Typhoon models) or **API** (Typhoon Cloud).
    * If using API mode, the system automatically enforces a 3.1s delay between files to stay within the 20 req/m safety limit.

3.  **Workflow**:
    * **Upload**: Drag and drop your invoice files.
    * **Process**: Click "Start Batch Processing".
    * **Verify**: Check the "System Check" status generated by the AI.
    * **Edit**: Correct any details in the table if the AI made a mistake.
    * **Approve**: Tick the "Approve?" checkbox for valid invoices.
    * **Export**: Download the final audit log as a CSV for your finance department.

## 🧠 System Logic & Safety

### Rate Limiting (Anti-Ban)
The system is optimized for the following Typhoon Cloud limits:
* **OCR**: 20 requests per minute.
* **LLM**: 200 requests per minute.

**Strategy**: The code implements a mandatory `time.sleep(3.1)` loop to ensure the OCR endpoint is never hit faster than the allowed rate. If a `429 Too Many Requests` error occurs, a 10-second "Cooldown" is triggered before a single retry.

### Structured Extraction
The extraction prompt is tuned to return high-precision JSON for seamless dataframe integration:
